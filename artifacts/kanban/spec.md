## Overview
할 일을 시각적으로 관리할 수 있는 Trello 스타일 칸반 보드. 3개 고정 칼럼에서 카드를 드래그&드롭으로 이동하고, 검색·필터·다크모드를 지원하며 localStorage로 데이터를 영속한다.

## 시나리오

### 1. 칸반 보드 초기 표시
[상황] 사용자가 앱을 처음 열면
[동작] 빈 상태의 3개 칼럼(Todo, In Progress, Done)이 가로로 나열된다

성공 기준:
- [ ] 앱 진입 → 칼럼 3개가 "Todo", "In Progress", "Done" 순서로 표시
- [ ] localStorage에 데이터가 없으면 → 각 칼럼에 카드 0개

### 2. 카드 생성
[상황] 사용자가 칼럼의 "추가" 버튼을 클릭하고 제목을 입력하면
[동작] 해당 칼럼 하단에 새 카드가 추가된다

성공 기준:
- [ ] Todo 칼럼에서 "추가" 클릭 → 제목 입력 필드 표시
- [ ] 제목 "회의록 작성" 입력 후 확인 → Todo 칼럼에 "회의록 작성" 카드 표시
- [ ] 제목 빈 문자열로 확인 → "제목을 입력해주세요" 에러 메시지 표시

### 3. 카드 제목 인라인 편집
[상황] 사용자가 카드의 제목을 클릭하면
[동작] 제목이 편집 모드로 전환되어 바로 수정할 수 있다

성공 기준:
- [ ] 카드 제목 "회의록 작성" 클릭 → 입력 필드로 전환, 현재 값 "회의록 작성" 표시
- [ ] 제목을 "주간 회의록 작성"으로 수정 후 확인 → 카드 제목 "주간 회의록 작성" 표시

### 3-1. 카드 상세 모달
[상황] 사용자가 카드를 클릭하여 모달을 열면
[동작] 모달에서 제목, 설명, 우선순위, 태그, 서브태스크를 모두 편집할 수 있다

성공 기준:
- [ ] 카드 클릭 → 상세 모달 열림
- [ ] 모달에서 제목 "회의록 작성"을 "주간 회의록 작성"으로 수정 → 제목 변경 반영
- [ ] 설명 영역에 "회의 안건 정리" 입력 후 저장 → 설명 "회의 안건 정리" 표시
- [ ] 우선순위 선택 → High / Medium / Low 선택지 표시
- [ ] 태그 영역에서 "긴급" 입력 후 확인 → "긴급" 태그 표시
- [ ] 모달 닫기 → 카드에 변경사항 반영

### 4. 카드 삭제
[상황] 사용자가 카드의 삭제 버튼을 클릭하면
[동작] 확인 다이얼로그 후 카드가 제거된다

성공 기준:
- [ ] 삭제 버튼 클릭 → "삭제하시겠습니까?" 확인 다이얼로그 표시
- [ ] 확인 클릭 → 해당 카드가 칼럼에서 사라짐
- [ ] 취소 클릭 → 카드 유지

### 5. 서브태스크(체크리스트) 관리
[상황] 사용자가 카드에 서브태스크를 추가하거나 체크하면
[동작] 서브태스크가 목록으로 표시되고 진행률이 반영된다

성공 기준:
- [ ] 서브태스크 "자료 준비" 추가 → 체크리스트에 "자료 준비" 항목 표시
- [ ] 서브태스크 2개 중 1개 체크 → 진행률 "1/2" 표시
- [ ] 서브태스크 삭제 버튼 클릭 → 해당 항목 제거

### 6. 카드 드래그&드롭
[상황] 사용자가 카드를 드래그하여 다른 칼럼이나 위치로 이동하면
[동작] 카드가 새 위치에 배치되고 순서가 저장된다

성공 기준:
- [ ] Todo의 카드를 In Progress로 드래그 → In Progress 칼럼에 카드 표시
- [ ] 같은 칼럼 내에서 카드 순서 변경 → 변경된 순서 유지
- [ ] 이동 후 페이지 새로고침 → 이동된 위치 그대로 표시

### 7. 제목 검색
[상황] 사용자가 검색창에 텍스트를 입력하면
[동작] 제목에 해당 텍스트가 포함된 카드만 표시된다

성공 기준:
- [ ] "회의" 입력 → 제목에 "회의"가 포함된 카드만 표시
- [ ] 검색어 삭제 → 전체 카드 다시 표시
- [ ] 매칭 카드 없음 → 빈 칼럼 표시

### 8. 우선순위·태그 필터
[상황] 사용자가 우선순위 또는 태그 필터를 선택하면
[동작] 조건에 맞는 카드만 표시된다

성공 기준:
- [ ] 우선순위 "High" 필터 선택 → High 카드만 표시
- [ ] 태그 "긴급" 필터 선택 → "긴급" 태그가 있는 카드만 표시
- [ ] 필터 해제 → 전체 카드 다시 표시

### 9. 다크모드 토글
[상황] 사용자가 다크모드 토글을 클릭하면
[동작] 전체 UI가 다크/라이트 테마로 전환된다

성공 기준:
- [ ] 초기 진입 시 시스템 설정에 따른 테마 적용
- [ ] 토글 클릭 → 반대 테마로 전환
- [ ] 새로고침 후 → 선택한 테마 유지

### 10. 데이터 내보내기
[상황] 사용자가 "내보내기" 버튼을 클릭하면
[동작] 전체 보드 데이터가 JSON 파일로 다운로드된다

성공 기준:
- [ ] 내보내기 클릭 → .json 파일 다운로드
- [ ] 파일 내용에 모든 칼럼과 카드 데이터 포함

### 11. 데이터 가져오기
[상황] 사용자가 JSON 파일을 가져오기하면
[동작] 기존 데이터를 교체하고 보드를 새로 표시한다

성공 기준:
- [ ] 유효한 JSON 파일 가져오기 → 기존 데이터 교체, 보드 갱신
- [ ] 잘못된 형식의 파일 → "올바른 형식이 아닙니다" 에러 표시

## 범위

### 포함
- 고정 3칼럼 칸반 보드 (Todo / In Progress / Done)
- 카드 CRUD (제목, 설명, 우선순위, 태그, 서브태스크)
- 카드 제목 인라인 편집
- 카드 상세 모달 (제목, 설명, 우선순위, 태그, 서브태스크 모두 편집)
- 드래그&드롭 (칼럼 간 이동 + 칼럼 내 순서 변경)
- 제목 검색 (실시간)
- 우선순위·태그 필터
- 다크모드 토글 (시스템 설정 기본값)
- localStorage 영속
- JSON 내보내기/가져오기 (전체 교체)

### 제외
- 칼럼 커스터마이징 (추가/삭제/이름변경) — 고정 3칼럼으로 충분, 복잡도 증가 대비 가치 낮음
- 사용자 인증/멀티유저 — 로컬 전용 앱
- 서버 연동/API — localStorage만 사용
- 카드 담당자 필드 — 싱글 유저 앱이므로 불필요
- 마감일/캘린더 — 1차 범위 외

## 전제 조건
- React + TypeScript 기반 SPA
- 브라우저 localStorage 사용 가능
- 모던 브라우저 (드래그&드롭 API 지원)
