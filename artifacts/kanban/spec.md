## Overview
Trello 스타일의 칸반 보드로 할 일을 관리하는 앱. 드래그&드롭으로 카드를 이동하고, 우선순위·태그·마감일·서브태스크로 작업을 체계적으로 추적한다.

## 시나리오

### 1. 카드 추가
[상황] 사용자가 칼럼에서 카드 추가 버튼을 클릭하고 제목을 입력하면
[동작] 해당 칼럼에 새 카드가 추가된다

성공 기준:
- [ ] 제목 "회의록 작성" 입력 후 추가 → Todo 칼럼에 "회의록 작성" 카드가 표시된다
- [ ] 빈 제목으로 추가 시도 → 카드가 추가되지 않고 에러 메시지가 표시된다

### 2. 카드 삭제
[상황] 사용자가 카드의 삭제 버튼을 클릭하면
[동작] 확인 대화상자가 표시되고, 확인 시 카드가 삭제된다

성공 기준:
- [ ] 삭제 버튼 클릭 → "정말 삭제하시겠습니까?" 확인 대화상자가 표시된다
- [ ] 확인 클릭 → 카드가 보드에서 사라진다
- [ ] 취소 클릭 → 카드가 그대로 유지된다

### 3. 카드 편집 (모달)
[상황] 사용자가 카드를 클릭하면
[동작] 모달이 열리고, 제목·우선순위·태그·마감일·서브태스크를 편집할 수 있다

성공 기준:
- [ ] 카드 클릭 → 편집 모달이 열린다
- [ ] 제목을 "주간 보고서"로 수정 후 저장 → 보드에 "주간 보고서"로 표시된다
- [ ] 우선순위를 "High"로 변경 → 카드에 High 색상이 적용된다

### 4. 드래그&드롭으로 칼럼 간 이동
[상황] 사용자가 카드를 다른 칼럼으로 드래그&드롭하면
[동작] 카드가 대상 칼럼으로 이동한다

성공 기준:
- [ ] Todo의 카드를 In Progress로 드롭 → 카드가 In Progress 칼럼에 표시된다
- [ ] 원래 칼럼(Todo)에서 해당 카드가 사라진다

### 5. 드래그&드롭으로 칼럼 내 순서 변경
[상황] 같은 칼럼 안에서 카드를 드래그하여 다른 위치에 드롭하면
[동작] 카드 순서가 변경된다

성공 기준:
- [ ] Todo 칼럼에 카드 A, B, C가 순서대로 있을 때, A를 C 아래로 드롭 → 순서가 B, C, A가 된다

### 6. 우선순위 색상 구분
[상황] 카드에 우선순위가 설정되어 있으면
[동작] 우선순위별로 다른 색상으로 카드가 표시된다

성공 기준:
- [ ] High 우선순위 카드 → 빨간 계열 색상으로 표시된다
- [ ] Medium 우선순위 카드 → 노란 계열 색상으로 표시된다
- [ ] Low 우선순위 카드 → 초록 계열 색상으로 표시된다

### 7. 태그 관리
[상황] 사용자가 카드 편집 모달에서 태그를 추가하면
[동작] 카드에 태그가 표시된다

성공 기준:
- [ ] "버그" 태그 추가 → 카드에 "버그" 태그가 표시된다
- [ ] 여러 태그("디자인", "기능") 추가 → 모두 카드에 표시된다

### 8. 마감일 설정 및 경고
[상황] 카드에 마감일이 설정되어 있고 마감일이 지났으면
[동작] 카드에 시각적 경고(빨간 테두리)가 표시된다

성공 기준:
- [ ] 마감일을 오늘 이전 날짜로 설정 → 카드에 빨간 테두리가 표시된다
- [ ] 마감일이 오늘 이후인 카드 → 경고 표시 없음

### 9. 서브태스크 관리
[상황] 사용자가 카드 편집 모달에서 서브태스크를 추가하고 토글하면
[동작] 서브태스크 체크리스트가 관리되고, 카드에 진행률이 표시된다

성공 기준:
- [ ] 서브태스크 "디자인 검토", "코드 리뷰" 추가 → 카드에 "0/2" 표시
- [ ] "디자인 검토" 체크 → 카드에 "1/2" 표시
- [ ] 서브태스크 삭제 후 1개 남음, 체크 상태 → 카드에 "1/1" 표시

### 10. 제목 검색
[상황] 사용자가 검색창에 텍스트를 입력하면
[동작] 제목에 해당 텍스트가 포함된 카드만 표시된다

성공 기준:
- [ ] "회의"로 검색 → 제목에 "회의"가 포함된 카드만 보인다
- [ ] 검색어 삭제 → 모든 카드가 다시 표시된다

### 11. 우선순위·태그 필터링
[상황] 사용자가 우선순위 또는 태그 필터를 선택하면
[동작] 해당 조건에 맞는 카드만 표시된다

성공 기준:
- [ ] "High" 우선순위 필터 선택 → High 카드만 보인다
- [ ] "버그" 태그 필터 선택 → "버그" 태그가 있는 카드만 보인다
- [ ] 필터 해제 → 모든 카드가 다시 표시된다

### 12. 다크모드 토글
[상황] 사용자가 다크모드 토글을 클릭하면
[동작] 테마가 전환되고, 새로고침 후에도 유지된다

성공 기준:
- [ ] 라이트 모드에서 토글 클릭 → 다크 모드로 전환된다
- [ ] 페이지 새로고침 → 다크 모드가 유지된다

### 13. 데이터 내보내기
[상황] 사용자가 내보내기 버튼을 클릭하면
[동작] 현재 보드 데이터가 JSON 파일로 다운로드된다

성공 기준:
- [ ] 내보내기 클릭 → JSON 파일이 다운로드된다
- [ ] 다운로드된 파일에 모든 칼럼과 카드 데이터가 포함되어 있다

### 14. 데이터 가져오기
[상황] 사용자가 JSON 파일을 가져오기하면
[동작] 기존 보드 데이터를 덮어쓰고 가져온 데이터로 복원된다

성공 기준:
- [ ] 유효한 JSON 파일 가져오기 → 보드가 해당 데이터로 교체된다
- [ ] 기존 카드가 모두 사라지고 가져온 카드만 표시된다

### 15. 데이터 영속성
[상황] 사용자가 카드를 추가/수정/이동한 후 페이지를 새로고침하면
[동작] 보드 상태가 그대로 유지된다

성공 기준:
- [ ] 카드 추가 후 새로고침 → 추가한 카드가 그대로 표시된다
- [ ] 칼럼 이동 후 새로고침 → 이동한 위치가 유지된다

## 범위

### 포함
- 3개 고정 칼럼(Todo, In Progress, Done) 칸반 보드
- 카드 CRUD (추가, 모달 편집, 삭제 with 확인)
- 드래그&드롭 (칼럼 간 이동 + 칼럼 내 순서 변경)
- 카드 속성: 제목, 우선순위(High/Medium/Low), 태그, 마감일, 서브태스크
- 제목 검색, 우선순위·태그 필터링
- 다크모드 토글 (localStorage 저장)
- JSON 내보내기/가져오기 (가져오기 시 덮어쓰기)
- localStorage 기반 데이터 영속성

### 제외
- 사용자 인증/로그인 — 단일 사용자 로컬 앱이므로 불필요
- 실시간 협업/멀티유저 — 로컬 스토리지 기반이므로 범위 밖
- 칼럼 추가/삭제/이름 변경 — 3개 고정 칼럼으로 범위 제한
- 카드 설명(본문) 필드 — 요구사항에 명시되지 않음
- 서버/DB 연동 — localStorage만 사용

## 전제 조건
- 단일 사용자가 하나의 브라우저에서 사용한다
- 칼럼은 Todo, In Progress, Done 3개로 고정된다
- 카드 추가 시 기본 우선순위는 지정하지 않아도 된다 (선택 사항)
